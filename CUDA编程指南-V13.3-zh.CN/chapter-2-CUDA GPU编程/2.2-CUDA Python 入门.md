# 2.2. CUDA Python 入门

本章介绍 Python 中的 CUDA 内核编程。CUDA Python 生态系统包含一个庞大且不断发展的工具和库生态系统。本章将首先介绍其中的一些组件，然后使用其中几个来说明在 Python 中编写和执行 GPU 代码的方法。

有很多种方法可以利用 GPU 进行计算，其中许多方法不需要显式编写 GPU 内核。[CUDA Python 生态系统](#221-cuda-python-生态系统) 的某些组件提供的函数在 GPU 上执行操作，而开发人员无需进行任何特定的 GPU 控制或编码。NVIDIA 加速计算中心提供了一个加速 Python 用户指南，其中介绍并讨论了许多不同的库和工具，这些库和工具支持在 Python 中进行 GPU 加速计算。对于希望尽可能快速轻松地在 Python 中使用 GPU 而不必直接编写 GPU 代码的用户来说，该资源是一个很好的起点。

另一方面，本章侧重于直接控制 GPU 并在 Python 中编写在 GPU 上执行的内核。本章重点介绍 Python 中的 CUDA 单指令多线程 (SIMT) 编程。

## 2.2.1. CUDA Python 生态系统

CUDA Python 是一个支持在 Python 中进行 GPU 计算的工具和库生态系统。以下列表介绍了 CUDA Python 的主要组成部分，并非所有部分都是此处涵盖的内容所必需的。此列表改编自 CUDA Python GitHub 存储库中的完整列表。

**主要组件** - 用于 GPU 控制和运行库提供的 GPU 代码
*   `cuda.core` - 用于 CUDA 控制（如内存和设备管理）的 Pythonic 接口。为 Python 提供了 CUDA 运行时为 CUDA C++ 提供的功能。
*   `cuda.compute` - 一个 Python 模块，提供由 CUDA 核心计算库 (CCCL) 提供的 GPU 加速函数。
*   `CuPy` - 一个 Python 库，提供 Numpy 例程的 GPU 加速版本，以及 `ndarray` 数据容器的 GPU 加速版本。

**内核编写组件**
*   `cuda.lang` - 一个 Python 领域特定语言 (DSL)，用于使用 Python 语言的子集在 SIMT 编程模型中编写 CUDA 内核和设备函数。
*   `cuda.coop` - 一个 Python 模块，提供 CUDA 核心计算库 (CCCL) 的可在设备上调用的原语（使用 `cuda.lang`）。
*   `cuda.tile` - 一个 Python 领域特定语言 (DSL)，用于在 Tile 编程模型中编写 CUDA 内核和设备函数。

**其他组件**
*   `cuda.pathfinder` - 用于定位 Python 环境中安装的 CUDA 组件的实用程序。
*   `cuda.bindings` - 用于 CUDA 库和实用程序（包括 CUDA 驱动 API、CUDA 运行时 API、NVRTC、NVVM 等）的低级 Python 绑定。`cuda.bindings` 通过 CUDA 驱动和 CUDA 运行时组件提供与 `cuda.core` 中相同的功能。但是，`cuda.bindings` 将这些功能作为 C 语言 API 的 Python 包装器提供，而不是原生的 Pythonic 接口。

### 2.2.1.1. 在 Python 中使用 CUDA 库

CUDA C++ 拥有丰富的库生态系统，这些库支持 GPU 加速，而无需直接编写内核或 GPU 代码。当 CUDA C++ 在 2006 年推出时，库很少，开发人员通常必须自己编写 GPU 内核代码。从那时起，已经开发了大量库，使开发人员能够在 C++ 中利用 GPU 计算，而无需编写太多（甚至不需要编写）GPU 代码。

CUDA Python 生态系统从另一个方向发展而来：诸如 CuPy 之类的 Python 库在能够直接使用 Python 语法和语义编写自定义内核之前，就为 Python 开发人员提供了计算和算法的 GPU 加速实现。这些库中的许多提供了对用 CUDA C++ 实现的 GPU 代码的 Python 绑定。

在现代 CUDA 时代，如果 GPU 加速库能满足您的需求，几乎总是建议使用它们。这些库中的许多提供了由 GPU 计算专家调整的实现。当库不可用或不够用时，可以直接在 Python 中编写 GPU 内核和设备函数，就像在 C++ 中一样。

### 2.2.1.2. 本章范围

虽然开发人员应尽可能优先使用库，但本章的其余部分将介绍当开发人员需要在 Python 中编写自定义 GPU 代码时如何进行。本章以与 C++ 的 [第 2.1 节](intro-to-cuda-cpp.html#intro-applications-cpp) 相同的方式介绍如何在 GPU 上编写和运行自定义 Python 代码，从如何指定 GPU 内核开始，然后如何使用 cuPy 提供的 GPU 加速 `ndarray` 在 GPU 上分配内存并提供 CPU 和 GPU 之间的通信。

### 2.2.1.3. 环境设置

通常，大多数 CUDA Python 生态系统组件都可以在 PyPi 上获得，并且可以使用 `pip` 命令或任何流行的 Python 包管理器进行安装。所有包都要求系统上安装了最新的 NVIDIA 驱动程序。编写或运行 CUDA Python 应用程序通常不需要 CUDA 工具包。

有关为不同平台安装和配置 CUDA Python 的信息，请参阅 NVIDIA 开发者专区上的 CUDA Python。

### 2.2.1.4. 运行 CUDA Python 应用程序

CUDA Python 应用程序，无论是使用 CUDA 加速库还是用户编写的 GPU 代码，都以与传统 Python 应用程序相同的方式运行。在本节中，示例将始终通过调用 `python3` 从命令行运行，如下所示，以执行名为 `cuda-python-app.py` 的程序。

```bash
$ python3 cuda-python-app.py
```

## 2.2.2. Python 中的 SIMT 内核

如 CUDA 编程模型引言中所述，在 GPU 上执行并可从主机调用的函数称为内核。CUDA 提供了两种不同的模型：单指令多线程 (SIMT) 和 CUDA tile。SIMT 内核的编写旨在由许多并行线程同时执行。这个概念在 CUDA Python 和 CUDA C++ 中是相同的。本章使用 SIMT 内核介绍 CUDA Python。

### 2.2.2.1. 指定内核

在 CUDA Python 中指定内核之前，必须导入包 `numba.cuda`。通常如下所示进行导入。

```python
from numba import cuda
```

这将导入 `numba.cuda` 包，并允许我们使用该包提供的 `cuda` 命名空间的组件。

要将一个函数指定为 CUDA Python 中的内核，请在该函数定义的上方放置装饰器 `@cuda.jit`，如下所示。

```python
from numba import cuda

@cuda.jit
def function(input_array, output_array):
    ...
```

这将导致内核在首次启动时为目标 GPU 进行 JIT 编译。当未指定其他 GPU 时，使用默认的 CUDA 设备，本节中显示的示例就是这种情况。

### 2.2.2.2. 启动内核

执行内核的线程数作为内核启动的一部分来指定。这称为执行配置。内核的每次调用都可以具有独特的执行配置，例如不同的块大小或线程块数量。

#### 2.2.2.2.1. 内核启动

要启动内核，执行配置放在内核名称之后、函数参数之前的方括号 `[ ]` 中。参数的顺序与 C++ 中的三括号表示法相同，具体如下：

```python
kernel_name[number_of_thread_blocks, threads_per_block](arguments, ...)
```

下面的代码片段展示了如何在 Python 源文件中定义然后调用内核。

```python
from numba import cuda

@cuda.jit
def my_kernel(input, output):
    ...

## launch the kernel
my_kernel[num_thread_blocks, threads_per_block](in_array, out_array)
```

每个线程块的线程数有限制，因为一个块的所有线程都位于同一个流式多处理器 (SM) 上，并且必须共享 SM 的资源。在当前 GPU 上，一个线程块最多可包含 1024 个线程。如果资源允许，多个线程块可以同时调度在一个 SM 上。

#### 2.2.2.2.2. 多维网格和线程块

线程块和线程块网格在 CUDA 中可以是 1 维、2 维或 3 维的。当网格或线程块是一维时，可以在指定内核启动的执行配置时使用整数。当线程块或网格是 2 维或 3 维时，使用二维或三维元组，如下面的二维网格和线程块启动所示，其中 `gridX` 和 `gridY` 是网格的 x 和 y 维度，`blockX` 和 `blockY` 是网格内每个线程块的 x 和 y 维度。

```python
from numba import cuda

@cuda.jit
def function(input, output):
    ...

## launch the kernel
function[(gridX, gridY), (blockX, blockY)](in_array, out_array)
```

### 2.2.2.3. 线程和网格索引内建变量

[第 1.2.2.1 节](../01-introduction/programming-model.html#programming-model-threads-grids) 介绍了线程和网格，[第 2.2.2.2 节](#intro-cuda-python-launching-kernels) 展示了如何为内核启动指定网格和线程块大小。在内核内部，每个线程可以访问执行配置的参数以及线程在网格中的索引和线程块索引。

可以从内核函数内部访问以下变量来确定线程的身份：

*   `cuda.threadIdx.[xyz]` 给出线程在其线程块内的索引。一个线程块中的每个线程将有不同的索引。
*   `cuda.blockDim.[xyz]` 给出线程块的维度，该维度在内核启动的执行配置中指定。
*   `cuda.blockIdx.[xyz]` 给出线程块在网格中的索引。每个线程块将有不同的索引。
*   `cuda.gridDim.[xyz]` 给出网格的维度，该维度在内核启动时的执行配置中指定。

这些变量中的每一个都是一个具有 `.x`、`.y` 和 `.z` 成员的三分量向量。如果在内核启动的执行配置中未指定某个维度，则对于维度，其默认值为 1；对于索引，其默认值为 0。

`cuda.threadIdx` 和 `cuda.blockIdx` 是从零开始的索引。也就是说，`cuda.threadIdx.x` 的取值范围是从 0 到 `cuda.blockDim.x - 1`（包括）。`.y` 和 `.z` 在其各自的维度上操作方式相同。

下面是一个简单的向量加法内核的代码，它将两个向量按元素相加。该函数接受三个数组 `A`、`B` 和 `C` 来实现按元素的向量加法 `C = A + B`。

```python
# C = A + B vector addition
@cuda.jit
def vecadd(A, B, C):
    idx = cuda.threadIdx.x + cuda.blockIdx.x * cuda.blockDim.x
    C[idx] = A[idx] + B[idx]
```

内核首先计算线程在网格中的唯一索引。此内核假设它是使用一维网格中的一维线程块启动的。`idx` 变量是从 0 到 `N-1` 的唯一索引，其中 N 是网格中的线程总数，即 `N = cuda.gridDim.x * cuda.blockDim.x`。

上面代码块中显示的计算线程索引的模式非常常见，Numba 为此操作提供了一种简写语法：`cuda.grid(n)`，其中 `n` 是维数。在上面的示例中，代码行

```python
idx = cuda.threadIdx.x + cuda.blockIdx.x * cuda.blockDim.x
```

可以替换为更简单的

```python
idx = cuda.grid(1)
```

此内核的一个值得注意的方面是它不检查对 `A`、`B` 或 `C` 的越界访问。在本章中，我们假设这些是由 cuPy 创建的 `ndarray`，稍后将在 [第 2.2.3.3 节](#intro-cuda-python-ndarray) 中介绍。当使用 cuPy `ndarray` 时，边界检查由数组类型隐式实现。

## 2.2.3. GPU 计算中的内存

> **注意**
> 像 CuPy 这样的 Python 包通过直接使用 CUDA C++ API（例如[第 2.1.3.2 节](intro-to-cuda-cpp.html#intro-cpp-explicit-memory-management)中介绍的 API）来执行 GPU 内存管理。多个 Python 包提供了用于控制 GPU 内存分配的包装器和实用程序。在本指南中，仅介绍 CuPy。大多数包的概念相似，并且除了特别说明外，其行为与 C++ 对应项类似。

如 [第 1.2.3 节](../chapter-1-CUDA简介/1.2-编程模型.md#123-gpu-memory-gpu内存) 所述，GPU 具有附加的 DRAM。在内核中使用的数据数组通常需要在内核访问之前位于 GPU 的 DRAM 中。在 Python 中，控制数据在内存中的位置，即在 CPU 和 GPU 之间移动数据，是程序员的责任。这与 C++ 中的显式内存管理的情况相同，如[第 2.1.3.2 节](intro-to-cuda-cpp.html#intro-cpp-explicit-memory-management)所述。

### 2.2.3.1. 在 GPU 上实例化数组

CuPy 提供了在 GPU 上创建指定类型和维度的 `ndarray` 对象，以及在 CPU 和 GPU 之间复制数据的函数。CuPy 中的许多函数具有与 Numpy 中创建 `ndarray` 的函数相似的函数签名。以下展示了如何使用 CuPy 在 GPU 内存中创建和填充数组的几个示例。

```python
import cupy as cp
import numpy as np

## create a matrix of zeros on the GPU
## when a datatype is not specified, float32 is used by default
A_device = cp.zeros((1024, 1024))

## create an array of 2^20 random doubles on the GPU
B_device = cp.rand.random((2**20), dtype=np.double)

## create an array of zeroes with the same shape and datatype as an existing array
C_device = cp.zeros_like(A)
```

### 2.2.3.2. 在主机和 GPU 内存之间复制数组

CuPy 也可用于将数据从驻留在 CPU 内存中的 Numpy `ndarray` 复制到驻留在 GPU 内存中的 CuPy 数组。

```python
import cupy as cp
import numpy as np

## Create an array in host memory
A_host = np.zeros((1024, 1024))
## Copy the array to the GPU
A_device = cp.array(A_host)

## Create an array in GPU memory
B_device = cp.rand.random((1024, 1024))
## copy the array to host memory
B_host = cp.asnumpy(B_device)
```

### 2.2.3.3. ndarray 对象类型

上一节中显示的 `ndarray` 对象存在于主机内存或 GPU 内存中，但不能同时存在于两者中。将驻留在主机上的数组作为参数传递给内核将导致错误。将驻留在 GPU 内存中的数组传递给普通的 Python 函数（即不是内核）也会导致错误。CuPy 不会隐式执行 CPU 和 GPU 之间的复制，因为复制开销可能很大，并且过多的数据复制会损害性能。因此，CuPy 要求程序员在数据何时在 CPU 和 GPU 之间复制时要明确。

在 GPU 内核中使用 `ndarray` 类型的一个优点是数组带有其维度的范围。如[第 2.2.2.3 节](#2223-线程和网格索引内建变量)所示，当所需线程总数略小于执行块或网格的大小时，边界检查会自动执行，内核代码无需检查越界访问。

## 2.2.4. 同步 CPU 和 GPU

与 C++ 一样，CUDA Python 中的内核启动相对于主机线程是异步的。也就是说，在内核启动之后，主机代码会在 CPU 上继续执行，而不保证内核已经完成甚至已经开始执行。为了保证 GPU 内核已完成执行，主机线程必须执行某种形式的与 GPU 的同步。

最简单的同步形式是同步整个 GPU。这种设备范围的同步是 CUDA 驱动程序提供的一种操作，并通过方法 `synchronize()` 向 Python 公开，cuPy 和 numba.cuda 都提供了该方法。

```python
import cupy as cp
from numba import cuda

..

## Wait on host thread for all pending GPU work to complete
## this uses the interface provided by cupy
cp.synchronize()

## Wait on host thread for all pending GPU work to complete
## this uses the interface provided by numba.cuda
cuda.synchronize()
```

设备范围的同步会使主机线程等待，直到 GPU 上所有先前发出的工作完成。使用 CUDA 流可以实现更细粒度的同步，如[第 2.5 节](./2.5-异步执行.md)所述。在 Python 中，当使用流时，建议的做法是使用 `cuda.core` 创建 CUDA 流，并根据需要仅与特定流执行同步。

## 2.2.5. 整合在一起

下面源代码清单中展示了执行并行向量加法的第一个 GPU 内核的 Python 形式。

```python
import numpy as np
from numba import cuda
import cupy as cp


## Defines a CUDA kernel to perform C = A + B vector addition
@cuda.jit
def vecadd(A, B, C):
    work_index = cuda.grid(1)
    C[work_index] = A[work_index] + B[work_index]


# note that vector size is not a power of 2 nor a multiple of the block_size defined below
vector_size = 2**24 + 11

device = cp.cuda.Device()
## Create device arrays of uniform random float32 values as input, and an array of zeros 
## as the result vector
a = cp.random.uniform(-1, 1, vector_size)
b = cp.random.uniform(-1, 1, vector_size)
c = cp.zeros_like(a)

block_size = 256
grid_size = int(np.ceil(vector_size/block_size))
vecadd[grid_size, block_size](a, b, c)

## synchronize the CPU thread and the GPU to ensure that the kernel has completed
## this is included to illustrate good practices, even though the copy below would implicitly wait for
## the kernel to complete
device.synchronize()

## Copy all 3 arrays to the CPU as ndarrays
a_np = cp.asnumpy(a)
b_np = cp.asnumpy(b)
c_np = cp.asnumpy(c)

## Perform the copy on the CPU to verify the answer
expected = a_np + b_np

## Test that the answer is correct, within floating point epsilon
np.testing.assert_array_almost_equal(c_np, expected)

## The assert will print diagnostics and abort
## so this only prints if the assertion passes
print("Test succeeded")
```

在此示例中，CuPy 在 GPU 上创建 `A` 和 `B` 输入数组并将其初始化为随机值。仅在代码末尾将它们复制到 CPU，以便 CPU 也可以执行向量加法并验证 CPU 和 GPU 的答案是否匹配。

## 2.2.6. CUDA Python 中的错误检查

任何影响 GPU 的操作，从内存分配和复制到内核启动，都可能导致错误情况。如 C++ 的[第 2.1.7 节](intro-to-cuda-cpp.html#intro-cpp-error-checking)所述，确保在与 GPU 交互的过程中没有发生错误是最佳实践。

在 Python 中，CUDA 错误会引发异常，如果不捕获，这些异常将终止程序。可以使用正常的 Python 语法捕获异常。下面的示例与上面的向量加法相同，但是有意添加了一个错误：每块线程数 2048 大于任何当前 GPU 可以运行的线程数。这将导致内核启动失败，并引发异常，此代码将捕获该异常。

```python
import numpy as np
from numba import cuda
import cupy as cp


## Defines a CUDA kernel to perform C = A + B vector addition
@cuda.jit
def vecadd(A, B, C):
    work_index = cuda.grid(1)
    C[work_index] = A[work_index] + B[work_index]


try:
    vector_size = 2**24 + 11

    device = cp.cuda.Device()
    a = cp.random.uniform(-1, 1, vector_size)
    b = cp.random.uniform(-1, 1, vector_size)
    c = cp.zeros_like(a)

    ## this block size is too large for any current GPUs
    block_size = 2048
    grid_size = int(np.ceil(vector_size/block_size))
    # Error: launching kernel with invalid block size
    vecadd[grid_size, block_size](a, b, c)

    device.synchronize()
    print("Test did not encounter any errors")

except Exception as e:
    print(f"Exception occurred: {e}")
```

运行此代码会导致错误被捕获并显示如下：

```bash
$ python3 vecadd_error.py
Exception occurred: CUDA_ERROR_INVALID_VALUE: This indicates that one or more of the parameters passed to the API call is not within an acceptable range of values.
```

程序正常退出，因为异常已被捕获。如果此代码在没有 `try:` 和 `except:` 的情况下运行，它将异常退出并将回溯信息转储到控制台，其中应显示相同的错误。
