# 1.1. 简介
## 1.1.1 The Graphics Processing Unit-图形处理单元（GPU）
图形处理单元最初作为一款专用于3D图形的处理器而诞生的，它最初是用来加速实时3D渲染中并行操作的功能固定的硬件。随着技术的发展，GPU的可编程性越来越强。到2003年，图形处理流程的某些阶段已经完全可编程，可以在3D场景或图像的每个部分并行运行自定义代码。

2006年，NVIDIA推出*Compute Unified Device Architecture(CUDA)-计算统一设备架构*，它使得任何计算工作负载都可以利用GPU的吞吐量能力，而不依赖图形API。

从此以后，CUDA和GPU计算已被用于加速所有类型的计算任务，从流体动力学或能量传输等科学仿真到数据库或分析等商业应用。此外，GPU的强大能力和可编程性为图像分类、生成式人工智能（扩散模型或大型语言模型）等各种新算法和技术的进步奠定了基础。

## 1.1.2. The Benefits of Using GPUs-使用GPU的优势
在相近的价格和功耗范围内，GPU的指令吞吐量和内存带宽远高于CPU。许多应用程序利用这些优势，在GPU上运行速度远超CPU。其他计算设备，像FPGA也具备很高的能效，但它编程灵活性比不上GPU。

GPU和CPU的设计目标各不相同。CPU的设计目标是尽可能快的高效执行一系列串行的操作（叫做线程），并且能够并行执行数十个这样的线程，而GPU的设计目标是并行执行数千个线程，它牺牲了单个线程的性能以换取更高的总吞吐量。

GPU专用于高度并行计算，并将更多晶体管用于数据处理单元，而CPU则将更多晶体管作为数据缓存和流控制。[图1](../assets/img/gpu-devotes-more-transistors-to-data-processing.png)展示了CPU和GPU芯片资源分配示例

## 1.1.3 Getting Started Quickly-快速开始
利用GPU提供的计算能力的方法有很多种。本指南涵盖了使用C++等高级语言为CUDA GPU平台进行编程的内容。然而，在应用程序中利用GPU的方式有很多，不需要直接编写GPU代码。

通过专门的库，可以获取来自各个领域的不断增长的算法和示例集合。如果某个库已经实现（特别是那些由NVIDIA提供的库），直接使用它往往比从头重新实现算法具有更高的效率和性能。像cuBLAS, cuFFT, cuDNN和CUTLASS等库只是众多帮助开发者避免重新实现成熟算法的几个例子。这些库的另一个优点是对每种GPU架构都进行了优化，达到生产力、效率和可移植性的理想平衡。

还有很多框架，特别是用于人工智能的框架，提供了GPU加速的构建模块。许多此类框架都是利用上诉的GPU加速库来实现加速的。

此外，像NVIDIA的Warp和OpenAI的Triron等领域特定语言(DSL)可以编译后直接运行在CUDA平台上。这提供了一种比本指南提到的高级语言更高层次的GPU编程方法。

[NVIDIA加速计算中心](https://github.com/NVIDIA/accelerated-computing-hub)包含了教授GPU和CUDA计算的资源、示例和教程。
