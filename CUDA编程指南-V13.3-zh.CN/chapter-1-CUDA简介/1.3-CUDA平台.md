# 1.3 The CUDA platform-CUDA平台
NVIDIA的CUDA平台由众多软件、硬件和许多重要的技术组成，这些技术是为了实现异构系统上的计算而开发的。本章主要介绍CUDA平台的一些基本概念和组成部分，它们对于应用开发人员理解CUDA平台至关重要。本章与<u>[Programming Model-编程模型](./1.2-编程模型.md)</u>类似，不针对任何编程语言，而是适用于所有使用CUDA平台的程序。

## 1.3.1 Compute Capability and Streaming Multiprocessor Versions-计算能力和流式多处理器版本
每款NVIDIA GPU都有一个*Compute Capability(CC)-计算能力*编号，该编号指示该GPU支持哪些特性，并且规定了该GPU的一些硬件参数。这些规范详见第<u>[5.1节]()</u>附录。NVIDIA GPU及其计算能力的列表维护在<u>[CUDA GPU Compute Capability page-CUDA GPU计算能力页](https://developer.nvidia.com/cuda/gpus)</u>。

计算能力以主版本号和次版本号表示，格式为X.Y，其中X是主版本号，Y是次版本号。例如，CC 12.0主版本号为12，次版本号为0。计算能力直接对应SM的版本号。例如，CC 12.0的GPU中的SM版本为sm_120，此版本用于标记二进制文件。

<u>[5.1.1小节]()</u>展示了如何查询和确定系统中GPU的计算能力。

## 1.3.2 CUDA Toolkit and NVIDIA Driver-CUDA工具包和NVIDIA驱动程序
*NVIDA Driver-NVIDIA驱动程序*可以看作是GPU的操作系统。NVIDIA驱动是一个软件包，必须安装在主机系统的操作系统上，并且是所有GPU使用（包括显示和图形功能）所必须的。NVIDIA驱动程序是CUDA平台的基础。除了CUDA，NVIDIA驱动还提供了使用GPU的所有其他方法，比如Vulkan和Direct3D. NVIDIA驱动的版本号像r580这样。

CUDA工具包是一套用于编码、构建和分析利用GPU计算的库、头文件、和工具。CUDA工具包是一个独立于NVIDIA驱动的软件产品。

*CUDA runtime-CUDA运行时*是由CUDA工具包提供的库中的一个特例。CUDA运行时提供了API和一些语言扩展，用于处理诸如内存分配、在GPU与其他GPU或CPU间复制数据以及启动kernel等常见任务。CUDA运行时的API组件统称为CUDA运行时API.

<u>[CUDA Compatibility-CUDA兼容性](https://docs.nvidia.com/deploy/cuda-compatibility/index.html)</u>文档提供了不同GPU、NVIDIA驱动和CUDA工具包版本间的完整详细信息。

### 1.3.2.1 CUDA Runtiom API and CUDA Driver API-CUDA运行时API和CUDA驱动API
CUDA运行时API是在名为CUDA驱动API的底层API之上实现的，而CUDA驱动API是由NVIDIA驱动公开的API。本指南重点介绍CUDA运行时API中公开的API。如果需要，所有相同的功能都可以仅使用驱动API来实现。有些功能仅可通过驱动API使用。应用程序可以使用任意一种API，也可以将两者互操作的同时使用。<u>[The CUDA Driver API]()</u>一节介绍了运行时API与驱动API之间的相互操作。

CUDA运行时API的完整API参考文档可以在<u>[CUDA Runtime API Documentation](https://docs.nvidia.com/cuda/cuda-runtime-api/index.html)</u>中找到。

CUDA驱动API的完整API参考文档可以在<u>[CUDA Driver API Documentation](https://docs.nvidia.com/cuda/cuda-driver-api/index.html)</u>中找到。

## 1.3.3 Parallel Thread Execution(PTX)-并行线程执行
CUDA平台的一个基础但有时不为人知的层叫做*Parallel Thread Execution(PTX) virtual instruction set architecture(ISA)-并行线程执行虚拟指令集架构*. PTX是一种用于NVIDIA GPU的高级汇编语言。PTX为实际GPU硬件的物理指令集架构ISA提供了一个抽象的层。与其他平台类似，可以直接用这种汇编语言编写应用程序，不过这样会给软件开发带来不必要的复杂性和难度。

*Domain-specific languages-领域特定语言*和高级语言编译器可以生成PTX代码作为中间表示(IR)，然后使用NVIDIA的离线或即时(just-in-time JIT)编译工具生成可执行的二进制GPU代码。这使得CUDA平台能够使用除NVIDIA提供的工具（如<u>[NVCC:The NVIDIA CUDA Compiler]()</u>）支持的语言外的其他语言进行编程。

由于GPU性能随着时间改变和进步，PTX虚拟指令集架构规范也采用了版本控制。PTX的版本与SM版本类似，PTX版本也对应了计算能力。例如，支持所有8.0计算能力特性的PTX被称为compute_80.

PTX的完整文档可以在<u>[PTX ISA](https://docs.nvidia.com/cuda/parallel-thread-execution/index.html)</u>找到。

## 1.3.4 Cubins(CUDA binary-CUDA二进制文件) and Fatbins
这俩就叫它英文名吧

CUDA应用程序和库文件经常使用C++等高级语言编写。这些高级语言会被编程成PTX文件，然后PTX文件会被编译成适用于物理GPU的实际二进制文件，称为CUDA二进制文件，简称*cubin*. cubin文件针对特定的SM版本(例如sm_120)具有特定的二进制格式。

用于GPU计算的可执行文件和库二进制文件同时含有CPU代码和GPU代码。GPU代码被存放在叫*fatbin*的容器中。Fatbins可以包含多个不同目标的cubins和PTX文件。例如，一个应用可以使用适用于不同GPU架构（即不同SM版本）的二进制文件构建。当应用程序运行，GPU代码被加载到指定的GPU上，并使用fatbin中最适合该GPU的二进制。

### 1.3.4.1 Binary Compatibility-二进制兼容性
NVIDIA GPU在特定情况下保证了二进制的兼容性。具体来说，在计算能力的主版本范围内，计算能力次版本大于或等于目标cubin的GPU可以加载和执行该cubin。例如，如果一个应用程序包含一个计算能力为8.6编译的cubin，该cubin可以在计算能力为8.6或8.9的GPU上加载和执行。但是，它不能加载到8.0计算能力的GPU上，因为GPU的计算能力次版本（0）低于代码的次版本号（6）。

NVIDIA GPU的主计算能力版本之间二进制不兼容。也就是说，在8.6计算能力上编译的cubin代码无法在计算能力9.0的GPU上加载。

当讨论二进制代码时，通常会用版本号表示，例如上例中的*sm_86*。这相当于说这个二进制文件是为计算能力8.6构建的。之所以经常用这种简写是因为开发者通常就是这样向NVIDIA CUDA编译器（nvcc）指定二进制构建目标的。

### 1.3.4.2 PTX Compatibility-PTX兼容性
GPU代码可以以二进制或者PTX的格式存储在可执行文件中，这在<u>[Cubins and Fatbins](#134-cubinscuda-binary-cuda二进制文件-and-fatbins)</u>中有详细介绍。当应用程序存储GPU代码的PTX版本时，该PTX代码可以在应用程序运行时进行JIT编译，以支持任何计算能力等于或大于PTX代码的环境。例如，如果应用程序包含用于*compute_80*的PTX代码，则可以在应用程序运行时将PTX代码JIT编译为更高版本的SM(如sm_120)。这实现了与未来GPU的前向兼容性，而无需重新构建应用程序或库。

### 1.3.4.3 Just-in-Time Compilation-即时编译
应用程序在运行时加载的PTX代码由设备驱动编译成二进制代码。这称为*just-in-time JIT-即时*编译。即时编译增加了应用程序的加载时间，但可以让应用程序得益于每个新设备驱动带来的编译器改进。它还使应用程序能够在编译时不存在的设备上运行。

当设备驱动给应用程序JIT编译PTX代码时，它会自动缓存生成的二进制代码副本，以避免在后续应用程序调用时重复编译。这个缓存被称为计算缓存，当设备驱动升级时它会自动失效，以便应用程序能够受益于设备驱动程序中内置的新型JIT编译器所带来的提升。

自CUDA最早版本以来，PTX在运行时进行JIT编译的方式和时间已经放宽。从而可以灵活的选择何时以及是否对部分或全部kernel进行JIT编译。<u>[Lazy Loading-懒加载]()</u>章节介绍了可用的选项以及如何控制JIT行为。此外，还有一些环境变量控制JIT编译行为，如<u>[CUDA Environment Variables-CUDA环境变量]()</u>中所述。

除了使用*nvcc*编译CUDA C++设备代码之外，NVRTC也可以用来在运行时将CUDA C++设备代码编译成PTX. NVRTC是一个用于CUDA C++的运行时编译库，更多信息请参阅NVRTC用户指南。
