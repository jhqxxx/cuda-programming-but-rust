# 安装 — cuda-oxide

本节将介绍在你的机器上运行 `cargo oxide run vecadd` 所需的一切。

---

## 前置条件
| 要求 | 版本 | 备注 |
|---|---|---|
| Linux | Ubuntu 24.04 已测试 | 其他发行版可能可用，但未经测试 |
| NVIDIA GPU | Ampere 架构及以上（sm_80+）| 驱动程序 545+ 推荐 |
| CUDA 工具包 | 12.x+ | `nvcc` 和 `cuda.h` 必须可用 |
| LLVM | 21+ | **必须包含 NVPTX 后端** |
| Clang | 21+ | `clang-21`——供 `bindgen` 为主机 `cuda-bindings` 使用 |
| Rust | Nightly（版本已固定） | 固定在 `rust-toolchain.toml` 中 |

> **注意**
> cuda-oxide 目前**仅支持 Linux**。不支持 Windows。

---

## 开发容器 (Dev Container)

代码库在 `.devcontainer/` 中包含了一个标准的 devcontainer 配置。使用它是最快的方式，可以获得一个可复现的开发环境，其中已预装 CUDA Toolkit 13.0、LLVM 21、Clang 21 和固定版本的 Rust nightly。

宿主机无需安装 CUDA Toolkit，但需要：

- 一块 NVIDIA GPU
- 与 CUDA 13.0 兼容的 NVIDIA 驱动
- 已安装 NVIDIA Container Toolkit 的 Docker

使用支持 devcontainer 的编辑器打开代码库，并在提示时选择"在容器中重新打开"。编辑器会读取 `.devcontainer/devcontainer.json`，构建镜像，使用 `--gpus=all` 请求 GPU 访问，并在容器内打开代码库。

仅使用命令行时，用以下命令启动容器：

```bash
npx -y @devcontainers/cli up --workspace-folder .
```

然后在容器内运行命令：

```bash
npx -y @devcontainers/cli exec --workspace-folder . cargo oxide doctor
npx -y @devcontainers/cli exec --workspace-folder . cargo oxide run vecadd
```

如果宿主机驱动版本太旧，容器内的 `nvidia-smi`、`cargo oxide doctor` 或 `cargo oxide run vecadd` 等 GPU 命令将会失败。请更新宿主机的 NVIDIA 驱动，而不是在容器内安装不同版本的 CUDA Toolkit。

如果你使用 devcontainer，可以跳过下面手动的 CUDA、LLVM、Clang 和 Rust 安装部分。

---

## CUDA Toolkit

从 [NVIDIA CUDA 下载页面](https://developer.nvidia.com/cuda-downloads) 安装 CUDA Toolkit，然后确保它在你的 `PATH` 中：

```bash
export PATH="/usr/local/cuda/bin:$PATH"
```

验证安装：
```bash
nvcc --version
```

> **提示**
> 如果你将 CUDA 安装到了非默认位置，请将 `CUDA_TOOLKIT_PATH` 设置为其根目录（包含 `include/cuda.h` 的目录）。如果未设置，cuda-oxide 默认使用 `/usr/local/cuda`。

---

## LLVM (21+)

cuda-oxide 使用 LLVM 的 NVPTX 后端将 LLVM IR 降级为 PTX。安装 LLVM 21 或更新版本，并确保 `llc-21`（或 `llc-22`）在你的 `PATH` 中：

```bash
# Ubuntu / Debian
sudo apt install llvm-21
```

如果你的发行版软件包不提供 `llvm-21`，请使用 LLVM 的 apt 辅助脚本：

```bash
sudo apt-get install -y lsb-release wget software-properties-common gnupg
wget https://apt.llvm.org/llvm.sh && chmod +x llvm.sh
sudo ./llvm.sh 21
```

验证 NVPTX 目标是否存在：

```bash
llc-21 --version | grep nvptx
```

你应该在已注册目标中看到包含 `nvptx64` 的一行。构建流程会自动按顺序发现 `llc-22` 和 `llc-21`；如需固定特定二进制文件，可设置 `CUDA_OXIDE_LLC=/usr/bin/llc-21`。

> **警告**
> 没有 NVPTX 后端的默认 LLVM 构建将无法工作。Ubuntu 的 `llvm-21` 软件包默认包含它，但如果你从源码构建 LLVM，必须向 CMake 传递 `-DLLVM_TARGETS_TO_BUILD="X86;NVPTX"`。

> **重要**
> **为什么需要 LLVM 21？** 我们生成的 TMA / tcgen05 / WGMMA 内联函数是 LLVM 20 及更早版本的 `llc` 无法处理的。简单的kernel可能仍能与旧版 `llc` 一起工作（设置 `CUDA_OXIDE_LLC=/path/to/llc-20`），但任何 Hopper / Blackwell 架构的代码都需要 21+。

## Clang（主机 `cuda-bindings`）

主机的 `cuda-bindings` crate 运行 `bindgen`，它在运行时加载 `libclang.so`，并需要 clang 自身的 resource-dir 中的 `stddef.h` —— 仅安装 `libclang1-*` 运行时是不够的。以下三个软件包涵盖了两部分：

```bash
sudo apt install libclang-21-dev libclang-cpp21-dev libclang-common-21-dev
```

`cargo oxide doctor` 会提前检测此问题；否则症状是在主机构建期间出现 `'stddef.h' file not found` 错误。

> **注意**
> **全新安装的 Ubuntu 24.04 / DGX-OS：** 通过上述 `apt.llvm.org/llvm.sh` 安装 LLVM 21 后，带版本号的 `clang-21` / `clang++-21` 二进制文件已存在，但 `cargo oxide doctor` 查找的无版本号别名尚未创建。使用 `update-alternatives` 添加它们：
> ```bash
> sudo update-alternatives --install /usr/bin/clang   clang   /usr/bin/clang-21   100
> sudo update-alternatives --install /usr/bin/clang++ clang++ /usr/bin/clang++-21 100
> ```
> 已在 Asus GX10 / NVIDIA DGX Spark 上验证。

## Rust 工具链

工作区附带了一个 `rust-toolchain.toml`，固定了确切的 nightly 版本和所需组件。当你在代码库中首次运行任何 `cargo` 命令时，`rustup` 会自动安装正确的工具链。

如需手动安装：

```bash
rustup toolchain install nightly-2026-04-03
rustup component add rust-src rustc-dev rust-analyzer --toolchain nightly-2026-04-03
```

这两个额外组件是代码生成后端所需的：

- `rust-src` —— Rust 标准库的源码，用于交叉编译到 NVPTX 目标。
- `rustc-dev` —— 后端链接所需的编译器内部结构。

---

## cargo-oxide

`cargo-oxide` 是驱动整个构建流程的 cargo 子命令（`cargo oxide run`、`build`、`debug`、`pipeline` 等）。

**在 cuda-oxide 代码库内**，它通过工作区别名即可直接使用 —— 无需额外安装步骤。

**在代码库外使用**（你自己的项目）：

```bash
cargo install --git https://github.com/NVlabs/cuda-oxide.git cargo-oxide
```

首次运行时，`cargo-oxide` 会自动获取并构建代码生成后端。后续运行将复用缓存的构建。

## 验证你的安装

运行内置的诊断检查：
```bash
cargo oxide doctor
```

`cargo oxide doctor` 会一次性验证你的 Rust 工具链、CUDA toolkit（包括 libNVVM / nvJitLink / libdevice，用于使用数学内联函数的kernel）、LLVM 安装和代码生成后端。

然后端到端地构建并运行一个示例：
```bash
cargo oxide run vecadd
```

如果一切配置正确，这将把一个 Rust kernel编译为 PTX，在 GPU 上启动它，并打印成功消息。

> **提示**
> **常见问题：**
> - `No working llc-21 or llc-22 found on PATH` —— 安装 LLVM 21+（`sudo apt install llvm-21`），将 `/usr/lib/llvm-21/bin` 添加到你的 `PATH`，或设置 `CUDA_OXIDE_LLC=/usr/bin/llc-21`。
> - 构建主机 `cuda-bindings` 时出现 `'stddef.h' file not found` —— 安装 clang 开发头文件：`sudo apt install clang-21`（或 `libclang-common-21-dev`）。
> - `cuda.h not found` —— 将 `CUDA_TOOLKIT_PATH` 设置为你的 CUDA 安装根目录，或确保 `/usr/local/cuda/include/cuda.h` 存在。
> - `rust-src component missing` —— 运行 `rustup component add rust-src --toolchain nightly-2026-04-03`。
